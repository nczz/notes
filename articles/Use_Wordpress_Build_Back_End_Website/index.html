使用 Wordpress 建置網站後台

Written by 莫小屁

前言

網站開發有前端後端，一般來說前端文案搭配視覺設計就是個好的開始，就像文宣一樣吸引人目光，不過因為有了資料動態需求，所以後端也要兼顧。

主題

有後台的網站大概會有以下需求：

  1. 最新消息
  2. 訪客留言
  3. 動態更新內容
  4. 內容分級
  5. …等等等

這些需求對會寫程式的人來說也不難，但是如果很常出現，每次都自己寫就累人了！ 網路上有很多的解決辦法，差別就是會不會用而已。

CMS系統 應該也是大家最常碰到的吧，像是 Blog、 論壇、 維基百科… 目前個人覺得資源最多彈性很夠的系統就屬 Wordpress 了！

拿它跟前端合作有幾招：

  1. 前端自己也 剛好 熟悉 WP 系統會改主題，後端調整系統配合前端 (優點: 這樣後端都快沒事了，缺點: 前端工作複雜化，搶後端飯碗XD)
  2. 後端將 WP 系統當 API ，盡全力配合前端設計 (優點: 前端專著視覺設計，設計與實做網站動態，缺點: 後端功能賣點變少)
  3. 請前端設計好畫面與操作動態流程，由後端將這一切給合併 (優點: 前端專著視覺設計，缺點: 後端包山包海)
  4. …等等等

這樣看來到底是要後端還是前端包山包海呢？ 這中間有很多可以討論的。

最完美的當然是兩方都可以辦到全包再來考慮切割吧…我想

有任何一方在技術背景不夠的時候都會造成另一方的問題，所以像我是前端弱就是盡全力支持前端

選擇 1 ： 要剛好熟悉的話可以從 這裡 開始

選擇 2 ： 要架好WP後安裝 deapress-json-api 這個外掛 PS: 還有很多很多的好外掛，只是要看需求去多嘗試找一個適合自己的

選擇 3 ： 把 WP 系統的架構摸熟，自己能夠改的得心應手

順便推薦其他最近測到還不錯的外掛：

  1. simple-image-sizes
  2. yet-another-related-posts-plugin
